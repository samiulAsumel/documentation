## TRAINING ARCHITECTURE — 12 PHASES

### PHASE 01 — Global Port System Foundation

**Scope:**
- What a port is as an economic and logistics system
- Port ecosystem structure (authority, terminal operator, shipping line, agent, customs)
- Cargo movement architecture end-to-end
- Global shipping chain: ship → port → inland logistics
- Port revenue generation logic (tariffs, storage, pilotage, crane hire)
- Why ports are the critical bottleneck in global trade

**Depth target:** Understand why a 4-hour delay at one terminal ripples into a 3-day schedule disruption across a shipping line's rotation.

---

### PHASE 02 — Vessel Operations

**Full lifecycle:**
- ETA notification → pre-arrival planning → pilot boarding → berthing → cargo operations → departure

**Core topics:**
- NOR (Notice of Readiness) — triggering conditions, disputes, time recording
- Laytime calculation — commencement, interruption, exceptions
- Tug coordination — number, type, bollard pull requirements
- Pilot boarding — boarding ground, VHF coordination, movement orders
- Berth allocation strategy — priority matrix, draft constraints, tide windows
- Vessel prioritization — feeder vs mainline vs reefer emergency

**Failure scenarios:**
- Vessel arrives 6 hours early — berth occupied, no tug available
- NOR tendered incorrectly — laytime dispute with shipping line
- Pilot unavailable during weather window
- Multi-vessel conflict: two vessels claiming same berth window

---

### PHASE 03 — Container Terminal Infrastructure

#### Yard Operations
- Bay–Row–Tier addressing system
- Stack optimization logic
- Yard zoning: import / export / transshipment / reefer / OOG / hazardous
- Dwell time control and escalation triggers
- Rehandle reduction strategy
- Pre-marshalling windows and sequencing logic

#### Crane Systems
- STS (Ship-to-Shore) crane operations
- RTG (Rubber Tyre Gantry) yard crane operations
- RMG (Rail Mounted Gantry) operations
- Crane productivity optimization: gross rate vs net rate vs utilization
- Equipment synchronization: STS → prime mover → RTG cycle

#### Gate Operations
- Truck appointment systems
- OCR (Optical Character Recognition) automation
- RFID gate processing
- Congestion management: truck queuing, gate split, appointment throttling

---

### PHASE 04 — Cargo Flow Engineering

**All flow types:**
- Import flow: vessel discharge → yard → customs → truck/rail release
- Export flow: booking → gate-in → yard → vessel load
- Transshipment: discharge → yard (or direct transfer) → load to connecting vessel
- Hub-and-spoke model mechanics
- Direct transfer vs yard transfer decision logic
- Coastal / cabotage logistics

**Bottleneck analysis:**
- Where flow breaks
- How congestion at one point creates upstream and downstream failure
- Quantifying cascade delay

---

### PHASE 05 — Port Billing Systems

**Tariff structures:**
- Port dues / vessel dues
- Pilotage and towage fees
- Crane hire / stevedoring charges
- Container handling charges (THC)
- Storage charges — free time, daily slabs, escalation tiers

**Critical logic:**
- Free time: definition, calculation, suspension conditions
- Slab billing: how storage escalates across time bands
- Demurrage vs detention vs port storage — exact definitions and liability
- Invoice breakdown analysis
- Billing dispute resolution process

**Financial scenarios:**
- 500 TEU import batch — 18-day dwell — slab billing calculation
- Disputed free time: vessel delay vs terminal fault — who pays?

---

### PHASE 06 — Terminal Operating System (TOS)

**Systems:**
- NAVIS / SPARCS N4
- COSMOS
- Custom TOS platforms

**Core modules:**
- Yard planning module
- Vessel planning module
- Gate module
- Billing and tariff integration

**EDI message types:**

| Message | Function |
|---------|----------|
| BAPLIE | Bay plan — container positions on vessel |
| COPRAR | Container pre-advice (loading instructions) |
| CODECO | Container departure/arrival confirmation |
| VERMAS | Verified gross mass declaration |
| COARRI | Container arrival report |

**Failure scenarios:**
- BAPLIE weight mismatch causing stowage recalculation
- TOS yard position conflict during crane operation
- EDI transmission failure — manual fallback procedures
- Billing module disconnection during peak shift

---

### PHASE 07 — KPI & Performance Engineering

**Primary KPIs:**

| KPI | Definition | Target Range |
|-----|------------|--------------|
| Gross crane rate | Moves per hour including non-productive time | 25–35 MPH |
| Net crane rate | Moves per hour during active lifting | 30–40 MPH |
| Berth productivity | Total moves / vessel time at berth | Vessel-class dependent |
| Yard utilization | TEU occupancy / total yard capacity | <85% operational limit |
| Dwell time | Days a container remains in terminal | Import <5d, TS <3d |
| Rehandle ratio | Non-productive moves / total moves | <8% target |
| Truck TAT | Gate-in to gate-out time | <45 minutes |
| Vessel TAT | First line to last line | Schedule dependent |

**For each KPI:**
- Root cause analysis framework
- Performance degradation triggers
- Corrective action sequence
- Impact on shipping line SLA

### PHASE 12 — Real-World Simulation Training

Five mandatory simulation scenarios. For each: full manager decision sequence, data requirements, operational response, KPI impact, and recovery timeline.

---

#### SIMULATION 01 — Early Vessel Arrival / Berth Conflict

**Scenario:** Vessel arrives 8 hours early. Berth occupied. ETA of departing vessel unknown. Two other vessels in anchorage waiting.

**Exercise:** Produce a berth prioritization decision with full justification, communication plan to all shipping line agents, and revised vessel sequence with estimated impact on laytime calculations.

---

#### SIMULATION 02 — Yard Saturation Crisis

**Scenario:** Yard at 94% utilization. Import containers dwelling 12+ days due to customs system failure. Export booking confirms 800 TEU for vessel arriving in 36 hours.

**Exercise:** Produce a yard recovery plan — identify containers to target for removal, coordinate with customs, prioritize export pre-marshalling, and calculate crane impact if yard is not cleared.

---

#### SIMULATION 03 — Crane Breakdown During Peak Operations

**Scenario:** STS crane #3 fails mid-operation. Vessel has 1,400 moves remaining. Next vessel due at berth in 11 hours. Only two operational cranes available.

**Exercise:** Recalculate completion time, renegotiate vessel ETA with agent, produce revised crane deployment plan, and document KPI impact on berth productivity and vessel TAT.

---

#### SIMULATION 04 — Customs System Failure Blocking Cargo Flow

**Scenario:** National customs portal down for 6 hours. 340 import containers cleared but system cannot generate release orders. 180 trucks queued at gate. Dwell clock running.

**Exercise:** Produce gate management contingency plan, communicate with customs authority, determine which containers can be released via manual authorization, and calculate storage revenue exposure.

---

#### SIMULATION 05 — Missed Transshipment Connection

**Scenario:** 220 TEU transshipment cargo missed connecting vessel due to crane breakdown. Next connecting vessel: 9 days. Shipping line demanding compensation and re-routing options.

**Exercise:** Produce a cargo disposition plan — identify re-routing via alternative feeder, calculate additional storage liability, draft communication to shipping line, and document root cause for operations review.
