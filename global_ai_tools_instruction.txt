# Technical Defaults — Linux / DevOps / Cloud

## Linux / RHCSA (RHEL 9)
- Default shell: bash; POSIX-portable unless bashisms add real value
- Use `systemctl`, `firewall-cmd`, `nmcli`, `timedatectl`, `semanage`, `restorecon`
- SELinux: enforcing mode assumed — always include context fixes
- Storage: LVM + XFS default; include persistent fstab/systemd mount units
- Prefer `ss` over `netstat`, `ip` over `ifconfig`, `journalctl` over raw logs

## DevOps / Cloud
- IaC: Terraform (HCL) preferred; annotate every resource block
- Containers: Podman-first on RHEL, Docker where explicitly stated
- CI/CD: GitHub Actions or GitLab CI; include lint/security scan stages
- AWS: IAM least-privilege; always include resource tagging strategy
- Azure: RBAC scoped to resource group minimum; managed identities, not keys
- Observability: Prometheus + Grafana + alerting rules

## Scripting
- Bash: strict mode always (`set -euo pipefail`), trap ERR, validate inputs
- Python: type hints, no bare `except`, use `pathlib` not `os.path`
- YAML/JSON: validate against schema before applying

# Technical Defaults — Web Design & Development

## Design Philosophy
- Every UI must have a **deliberate, committed aesthetic direction** — never generic
- Avoid: Inter/Roboto/Arial, purple gradients, cookie-cutter layouts, AI-slop patterns
- Choose a tone and execute with precision: brutalist, editorial, luxury, industrial,
  retro-futuristic, etc. — make it UNFORGETTABLE
- One clear visual concept beats ten mediocre ideas

## Frontend Stack (Modern Defaults)
- **Framework**: Next.js 14+ (App Router) or React 18+ with Vite
- **Styling**: Tailwind CSS v3+ with custom design tokens; or vanilla CSS with custom 
  properties — never inline styles at scale
- **Animations**: Framer Motion (React), GSAP for complex timelines, CSS-only for 
  micro-interactions
- **State**: Zustand or React Context for local; TanStack Query for server state
- **Forms**: React Hook Form + Zod validation — never uncontrolled forms in production
- **Icons**: Lucide React or Phosphor — never FontAwesome bloat

## Typography Rules
- Always pair a distinctive **display font** with a refined **body font**
- Source from: Google Fonts, Fontsource, or self-hosted for performance
- Define a modular type scale with CSS variables (`--text-xs` through `--text-7xl`)
- Line-height and letter-spacing tuned per weight — never defaults

## Color & Theme
- CSS custom properties for ALL colors — never hardcoded hex in components
- Dominant color + sharp accent + neutral base — avoid flat, even distributions
- Dark mode: implement via `prefers-color-scheme` + manual toggle from day one
- Contrast ratios: WCAG AA minimum, AAA for body text

## Layout & Composition
- CSS Grid for macro layout, Flexbox for component-level alignment
- Asymmetry, overlap, and grid-breaking elements where appropriate
- Generous negative space OR controlled density — never in between
- Mobile-first breakpoints: `sm:640 md:768 lg:1024 xl:1280 2xl:1536`

## Motion & Interaction
- Page load: staggered reveals with `animation-delay` — one well-orchestrated sequence
- Hover states that surprise; scroll-triggered animations for key sections
- Respect `prefers-reduced-motion` — always include the media query override
- No animation without purpose; every motion communicates something

## Backgrounds & Visual Depth
- Gradient meshes, noise textures, geometric patterns over flat colors
- Layered transparencies, dramatic shadows, grain overlays for atmosphere
- SVG patterns and clip-paths for structural visual elements

## Backend / API
- **Node.js**: Express or Fastify with typed routes; never callback hell
- **Python**: FastAPI with Pydantic models; async-first
- **Auth**: JWT + refresh tokens or NextAuth.js — never roll custom crypto
- **Database**: PostgreSQL default; Prisma or Drizzle ORM; migrations in version control
- **API design**: REST with OpenAPI spec OR tRPC for full-stack TypeScript
- Rate limiting, input sanitization, CORS policy — included by default, never optional

## Performance Standards
- Lighthouse score target: ≥90 Performance, 100 Accessibility, 100 Best Practices
- Images: next/image or `<img loading="lazy">` with explicit width/height always
- Fonts: `font-display: swap`; preload critical fonts
- Bundle: code-split at route level; analyze with `vite-bundle-visualizer`
- Core Web Vitals: LCP <2.5s, CLS <0.1, INP <200ms

## Security (Web)
- CSP headers configured; no `unsafe-inline` without explicit justification
- HTTPS enforced; HSTS preload where applicable
- Sanitize all user input server-side — client validation is UX, not security
- No secrets in frontend code, `.env` files, or git history — ever
- Dependency audit: `npm audit` in every CI pipeline

## Accessibility
- Semantic HTML first — ARIA only when native semantics are insufficient
- All interactive elements keyboard-navigable with visible focus styles
- Screen reader tested landmarks: `<main>`, `<nav>`, `<header>`, `<footer>`
- Color is never the sole means of conveying information
